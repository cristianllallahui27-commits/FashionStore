PROMPT 1: REQUISITOS DEL CLIENTE
─────────────────────────────────────────────────────────────────────────
"Necesito un sistema web administrativo para mi tienda de ropa y lencería
que permita:

1. GESTIÓN DE INVENTARIO
   - Crear, editar, eliminar prendas
   - Categorías de productos
   - Control de stock
   - Precios y descuentos

2. SISTEMA POS (PUNTO DE VENTA)
   - Crear ventas con múltiples items
   - Validar disponibilidad de stock
   - Aplicar descuentos
   - Procesar pagos (Efectivo, Tarjeta, etc.)
   - Generar recibos

3. GESTIÓN DE PERSONAL
   - Crear vendedores
   - Asignar contraseñas
   - Control de acceso (permisos)
   - Cambio de contraseña seguro

4. GESTIÓN DE CLIENTES
   - Registro de clientes
   - Búsqueda rápida
   - Historial de compras
   - Cliente mostrador (genérico)

5. REPORTES Y ANÁLISIS
   - Ventas por período
   - Ingresos totales
   - Productos más vendidos
   - Reporte por vendedor
   - Exportar a PDF/Excel

6. SEGURIDAD
   - Autenticación de usuarios
   - Roles de acceso
   - Auditoría de operaciones
   - Contraseñas hasheadas"

PROMPT 2: STACK TECNOLÓGICO ELEGIDO
─────────────────────────────────────────────────────────────────────────
Backend:
  • ASP.NET Core 9.0 (Framework MVC)
  • Entity Framework Core 9.0 (ORM)
  • PostgreSQL vía Supabase (BD)
  • Serilog (Logging)
  • xUnit (Testing)
  • AutoMapper (Mapping)

Patrones de Diseño:
  • Repository Pattern (Abstracción de datos)
  • Unit of Work (Transacciones)
  • Dependency Injection (IoC)
  • DTO Pattern (Transfer objects)

Frontend:
  • Razor Pages (Generación HTML)
  • Bootstrap 5 (Estilos)
  • JavaScript vanilla (Interactividad)
  • Chart.js (Gráficos)

Seguridad:
  • ASP.NET Identity (Autenticación)
  • Role-based Authorization
  • CSRF Tokens
  • Password Hashing (bcrypt via Identity)

PROMPT 4: MODELO DE DATOS (ENTIDADES)
─────────────────────────────────────────────────────────────────────────

TABLA: Categoria
├─ Id (PK)
├─ Nombre (Required, String 100)
└─ Descripcion (String 500, nullable)

TABLA: Prenda
├─ Id (PK)
├─ Nombre (Required, String 150)
├─ Descripcion (String 500)
├─ Precio (Decimal 10,2)
├─ Stock (Int >= 0)
├─ CategoriaId (FK)
└─ FechaCreacion (DateTime)

TABLA: Cliente
├─ Id (PK)
├─ DNI (String 8, Required)
├─ NombreCompleto (String 150)
├─ Email (String, nullable)
├─ Telefono (String 15)
└─ Direccion (String 300)

TABLA: Vendedor
├─ Id (PK)
├─ Nombres (String 150, Required)
├─ Apellidos (String 150, Required)
├─ DNI (String 8, Required)
├─ Correo (Email)
├─ Telefono (String 15)
├─ Estado (Boolean)
└─ [NotMapped] UsuarioId

TABLA: MetodoPago
├─ Id (PK)
└─ Nombre (String 100, Required)

TABLA: DescuentoAutorizado
├─ Id (PK)
├─ Nombre (String 100)
├─ Tipo (Enum: Porcentaje|Monto)
├─ Valor (Decimal)
├─ Activo (Boolean)
└─ FechaCreacion (DateTime)

TABLA: Venta
├─ Id (PK)
├─ Fecha (DateTime)
├─ Total (Decimal 10,2)
├─ Descuento (Decimal 10,2)
├─ VendedorId (FK)
├─ ClienteId (FK)
├─ MetodoPagoId (FK)
├─ MontoRecibido (Decimal, nullable - para efectivo)
├─ Vuelto (Decimal, nullable)
└─ DescuentoAutorizadoId (FK, nullable)

TABLA: DetalleVenta
├─ Id (PK)
├─ VentaId (FK)
├─ PrendaId (FK)
├─ Cantidad (Int > 0)
├─ Precio (Decimal 10,2)
└─ Subtotal (Decimal 10,2)

TABLA: AspNetUsers (Identity)
├─ Id (PK)
├─ Email
├─ UserName
├─ PasswordHash (Hasheado con bcrypt)
├─ Activo (Boolean)
└─ [Relations] Roles, Claims, Tokens

PROMPT 5: FLUJOS DE NEGOCIO PRINCIPALES
─────────────────────────────────────────────────────────────────────────

FLUJO A: CREAR VENDEDOR Y ASIGNAR ACCESO
─────────────────────────────────────────
Paso 1: Admin → Admin Menu → Vendedores
Paso 2: Click "Crear Vendedor"
Paso 3: Ingresar:
   - Nombres: "Ana"
   - Apellidos: "Usuario"
   - DNI: "82030000"
   - Email: "ana@fashionstore.com"
   - Contraseña: "Password123!" (Admin la asigna)
Paso 4: Sistema:
   - Crea ApplicationUser en Identity
   - Asigna rol "Vendedor"
   - Crea registro en tabla Vendedor
   - Sincroniza Correo = Email
Paso 5: Vendedor recibe Email + Contraseña
Paso 6: Vendedor inicia sesión
   - URL: http://localhost:5100/Identity/Account/Login
   - Email: ana@fashionstore.com
   - Contraseña: Password123!
Paso 7: Dashboard carga (rol: Vendedor)
   - NO ve menu Admin
   - Ve Ventas, Inicio

FLUJO B: REGISTRAR VENTA (POS)
─────────────────────────────────────────
Paso 1: Vendedor → Ventas → Nueva Venta
Paso 2: Sistema llena:
   - Vendedor: "Ana Usuario" (auto-detectado de sesión)
   - Fecha: Hoy
Paso 3: Vendedor selecciona Cliente:
   - Buscar: "Mostrador" (genérico)
   - O crear rápido: Nombre, DNI, Email
Paso 4: Agregar productos:
   - Click: "+ Agregar producto"
   - Seleccionar: "Blusa Rosa" (Categoría: Blusas)
   - Cantidad: 2
   - Precio unitario: 89.90 (auto)
   - Subtotal: 179.80 (auto-calculado)
Paso 5: Repetir para más productos
Paso 6: Aplicar descuento (si solo Admin):
   - Tipo: Seleccionar descuento autorizado
   - O Manual (solo Admin): % o Monto
Paso 7: Seleccionar método de pago:
   - Efectivo
   - Tarjeta
   - Depósito
Paso 8: Si Efectivo:
   - Monto a pagar: $179.80
   - Monto recibido: $200.00
   - Vuelto: $20.20 (auto-calculado)
Paso 9: Click "Registrar Venta"
Paso 10: Sistema:
   - Valida stock de cada producto
   - Calcula total
   - Decuenta stock de BD
   - Crea registro Venta + DetalleVenta
   - Registra en auditoría
Paso 11: Mensaje: "✓ Venta registrada exitosamente"
Paso 12: Opción: Descargar recibo (PDF)

FLUJO C: VER REPORTES (Admin)
─────────────────────────────────────────
Paso 1: Admin → Reportes
Paso 2: Filtros:
   - Rango de fechas
   - Vendedor (opcional)
   - Estado: Todas / Completadas
Paso 3: Click "Buscar"
Paso 4: Sistema muestra:
   - Tabla de ventas
   - Gráfico de ingresos
   - Top 5 productos
   - Total ventas: $5,234.50
   - Cantidad operaciones: 45
Paso 5: Opciones:
   - "Descargar Excel" → archivo .xlsx
   - "Descargar PDF" → archivo .pdf

FLUJO D: GESTIONAR DESCUENTOS (Admin)
─────────────────────────────────────────
Paso 1: Admin → Descuentos
Paso 2: Click "Crear Descuento"
Paso 3: Ingresar:
   - Nombre: "Black Friday 2026"
   - Tipo: Porcentaje
   - Valor: 20 (%)
   - Activo: Sí
Paso 4: Click "Crear"
Paso 5: En próximas ventas:
   - Admin puede seleccionar este descuento
   - Sistema aplica automáticamente
